## 2. transfer_money 槽位契约修正

### 2.1 新的业务口径

转账意图当前按以下槽位执行：

- 必填：`amount`
- 必填：`payee_name`
- 必填：`payee_card_no`
- 可选：`ccy`
- 可选：`payer_card_no`
- 可选：`payer_card_remark`
- 可选：`payee_card_remark`
- 可选：`payee_card_bank`
- 可选：`payee_phone`

### 2.2 本次修正点

此前实现里把 `payee_card_no` 折叠成“收款对象标识”，既可能放姓名，也可能放卡号，这不满足业务语义。

本次已拆分并收紧为：

- `payee_name` 只承载收款人姓名
- `payee_card_no` 只承载收款卡卡号/尾号
- `amount + payee_name + payee_card_no` 三个同时满足才允许完成转账

## 4. file 模式部署链路修正

router 当前运行在 file catalog 模式，但部署前仍会从 sqlite 导出 catalog。

为避免 repo 内最新 finance intent 定义未同步到 sqlite 就被旧快照覆盖，本次新增：

- `scripts/sync_financial_intents_to_db.py`

部署脚本 `scripts/minikube_deploy_intent.sh` 现在顺序为：

1. 先执行 `sync_financial_intents_to_db.py`
2. 再执行 `export_router_intent_catalog_from_db.py`
3. 再更新 ConfigMap 与 Deployment

这样部署出来的 file catalog 与当前 transfer source-of-truth 保持一致。
